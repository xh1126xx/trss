# 同声传译桌面应用 设计文档

## 概述

一个轻量 Windows 桌面应用，捕获系统音频，通过用户配置的多模态 LLM API 实时翻译，以悬浮字幕形式显示。

## 技术栈

- **框架**: Go + Wails
- **前端**: HTML/CSS/JS（Wails WebView）
- **音频**: WASAPI Loopback（Windows Core Audio）
- **目标体积**: ~10-15MB，内存 ~30-50MB

## 架构

```
┌─────────────────────────────────┐
│         Go 后端                  │
│                                 │
│  WASAPI 音频捕获 → 音频缓冲       │
│                    │            │
│                    ▼            │
│           ┌──────────────┐      │
│           │ 通用翻译客户端  │      │
│           └──────────────┘      │
│                                 │
└─────────────┬───────────────────┘
              │ Wails Runtime
┌─────────────▼───────────────────┐
│         Web 前端                 │
│                                 │
│  字幕悬浮窗  │  设置面板          │
│  系统托盘    │  配置方案管理      │
└─────────────────────────────────┘
```

应用本身不内置任何翻译服务商，所有 API 配置由用户自行填写。

## 模块设计

### 1. 音频捕获模块

- 通过 Windows WASAPI Loopback 捕获系统播放的所有音频
- 重采样为 16kHz 单声道 PCM
- 分帧（50-100ms）写入环形缓冲区

### 2. 通用翻译客户端

**核心接口：**

```go
type TranslatorConfig struct {
    BaseURL    string // API 地址
    APIKey     string // API Key
    Model      string // 模型名称
    SourceLang string // 源语言
    TargetLang string // 目标语言
    Prompt     string // 系统提示词
}

type Translator interface {
    Translate(ctx context.Context, audio []byte) (*Result, error)
    SetConfig(TranslatorConfig) error
}
```

**设计原则：**
- 零预置服务商，完全由用户配置
- 支持 OpenAI 兼容协议
- 支持用户自定义 API 地址（兼容中转/代理/自部署）
- 提示词自由编辑，支持 `{source}` `{target}` `{context}` 变量

**配置方案管理：**
- 用户可保存多套配置方案（不同 API、不同语言对、不同翻译风格）
- 提供方案切换、新建、删除功能
- 方案数据持久化到本地 JSON

### 3. 前端悬浮字幕窗

**窗口特性：**
- 透明窗口，仅字幕条可见
- 始终置顶（always-on-top）
- 穿透点击，不干扰下方窗口操作
- 位置可拖拽调整，默认屏幕底部居中

**三种显示模式：**

| 模式 | 显示内容 |
|------|----------|
| 仅翻译 | 只看译文 |
| 双语 | 原文 + 译文 |
| 仅原文 | 只看原文 |

**流式显示：**
- 临时结果半透明显示
- 最终结果加亮确认
- 每条字幕停留 3-5 秒后淡出

**样式可配置：**
- 字体大小
- 文字颜色
- 背景透明度
- 最大显示行数

### 4. 设置面板

独立窗口，分三个标签页：

| 标签页 | 内容 |
|--------|------|
| 翻译配置 | API 地址、Key、模型名、语言选择、提示词编辑、测试连接 |
| 显示 | 字体大小、颜色、透明度、背景、显示行数、字幕位置 |
| 音频 | 输入设备选择、采样率 |

### 5. 系统托盘

- 托盘图标 + 运行状态指示
- 右键菜单：开始/暂停、显示模式切换、打开设置、退出

## 数据流

```
系统音频
    │
    ▼
WASAPI Loopback (50ms 分帧)
    │
    ▼
音频缓冲 (环形缓冲区)
    │
    ▼
通用翻译客户端 (HTTP/WebSocket → 用户配置的 API)
    │
    ▼
流式返回翻译文本
    │
    ▼
字幕窗口显示 (临时→最终→淡出)
```

## 用户工作流

1. 启动应用（后台静默运行，托盘显示）
2. 打开设置 → 填写 API 配置（地址、Key、模型名）
3. 编辑提示词（或使用默认模板）
4. 选择源语言和目标语言
5. 测试连接确认可用
6. 打开视频/音频 → 开始监听
7. 字幕自动悬浮显示

## 多配置方案示例

用户可以创建多套方案用于不同场景：

- "英文→中文 (GPT-4o)" - 日常看英文视频
- "日文→中文 (Gemini)" - 看日剧/动漫
- "中文→英文 (通义千问)" - 和外国朋友分享中文内容
- "会议直译" - 自定提示词偏正式风格

## 技术依赖

| 依赖 | 用途 |
|------|------|
| Wails v3 | 桌面应用框架 |
| go-wca | Windows Core Audio 绑定 |
| Go 标准库 net/http | HTTP 客户端调用 API |

## 测试策略

- 音频捕获：验证 WASAPI Loopback 正确获取系统音频并分帧
- 翻译客户端：Mock API 验证请求格式和结果解析
- 前端字幕窗：验证窗口置顶、穿透点击、三种显示模式
- 配置管理：验证方案保存/切换/删除和数据持久化
